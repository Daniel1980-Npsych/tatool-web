Additional Steps

You've created your first Executable with custom logic that is reading a stimuli file and image stimuli, presents them on the screen and allows the user to provide a response. After processing the respone, the trial data is stored in the database. This is where our first tutorial ends but you're not done yet. Check out the following pages to learn about additional things you could do.

  • Add a Timer to reduce the display time of a Stimulus.
  • Randomise your stimuli by using the Stimuli Processing Methods of the executableUtils Service.
  • Create a different Stimuli File.
  • Create a Complex Span Executable and combine the two with a Dual Element.
  • Start styling your display by introducing your own CSS styles
