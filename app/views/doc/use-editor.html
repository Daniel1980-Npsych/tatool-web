Editor

The Editor is a convenient way to quickly and easily create experiments. The Editor interface consists of two parts, the module navigation on the left side and the module/element settings on the right side. Move your mouse over the Tooltip icons to get brief descriptions of each field.

Module Settings

The module setting comprise the basic module information (name, author, label), and the module behaviour settings. All module configurations are stored in JSON format. To easily exchange modules with other researchers, you can download a JSON file by clicking on the download button. The module that you create with the Editor, is saved as a JSON file. To download the JSON module file, click on the download button.

Elements and Handlers

Elements (List or Dual Elements) group your Executables and can be executed for n iterations. Handlers are also set on the level of elements, which makes their functionality available to any child Executables or elements (e.g., another List Element). To see what properties can be set for the handlers, have a look at the handlers reference.

Executables

Executables can be understood as experimental paradigms. They are hence the most important component of your experiment. Browse our Task Library to see which tasks are available. All of these tasks allow you to use your own stimuli, and most of them are so highly generic that you can tailor them to your needs tweaking their properties. The properties are listed for each task in the Task Library.

You can also develop your own tasks. If you're willing to share your Executables with other researchers, we'll be happy to host them for you for free! Otherwise, you can either run Tatool in offline lab mode (see here), host in on your own web server running Tatool, or we can host it for you in exchange for a small donation helping to cover our fixed hosting costs.

Publish

To go live with your experiment, you can publish it with a simple click on Publish (make sure you activated the Repository). You can choose whether you want to make the module available to all registered Tatool users by selecting it to be Public, or to invite only a specified sample of participants that are personally invited by selecting it to be Private. You can Edit your module any time, but have to Publish it again to make changes available to your users. Once you published such changes, users that have added your module first have to update it before they can execute it. This allows for dynamic adjustment as it might be required during piloting.

Note: To make sure that we have sufficient web space for everyone, we currently limit the number of simultaneously published modules per researcher to 3 at a time.
