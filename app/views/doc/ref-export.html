Data Export

Tatool allows you to configure your Module with different types of Exporters for different scenarios. An Exporter always extracts the underlying trial data of a Module and creates a CSV file. The structure of the CSV file always follows the same structure whereas every Trial you save in your Executable corresponds to one line and every property you add to a Trial corresponds to a column. There is a defined set of columns which will always be present.

CSV File Structure

The following attributes are added automatically to your Trials and are visible in the first columns of the export.

  • userCode: the internal Tatool user code which is unique for every user and allows you to identify the Trials of a specific user.
  • moduleId: the module generating the Trials. Corresponds to the Module Label provided in the Module Editor.
  • sessionId: the internal Tatool sequence for Sessions which allows you to identify Trials belonging to the same Session.
  • trialId: the internal Tatool sequence for Trials
  • executableId: the name of the Executable generating the Trials.

If any Executable or Handler stores Module Properties, they will be available in the following columns with the prefix module. For example a Module Property with the name lastValue as part of the Executable digitTask will receive a column header of module.digitTask.lastValue.

If any Executable or Handler stores Session Properties, they will be available in the following columns with the prefix session. For example a Session Property with the name lastValue as part of the Executable digitTask will receive a column header of session.digitTask.lastValue.
Tatool stores one mandatory session property with the column header session.complete which indicates whether the Session was completed successfully (1), e.g. all trials executed or Module successfully stopped by pressing Escape, or not completed due to an error (0), e.g. abort of a running Session by refreshing the page or closing the browser window/tab.

All the following columns are customized according to the properties you define in your Trial object. The column header naming will be [executableName].[propertyName]. For example an executable property named setSize as part of an Executable digitTask will receive a column header of digitTask.setSize.

Exporter Types

A Module can have multiple Exporters defined at the same time but only one of each type. The following Exporters are supported by Tatool:

  • Upload: automatically exports data and uploads it to the Tatool Server running the Module after every Session.
  • Download: allows the user to download all data to his local disk. (currently not supported with Safari)
