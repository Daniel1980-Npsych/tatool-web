Record Input

This section will introduce on how to capture user input by listening to keyboard input or mouse click. We're going to extend our previous code with some additional logic in the template, service and controller.

As we want to be able to display keys/buttons on screen we're going to make use of the tatool-input element which allows us to display keys/buttons/textfields as a method of user interaction and trigger some action at the time of user input. Take a look at the extended template below.

myExecutable.html

<tatool>
  <tatool-stimulus id="centeredExecutable" service="stimulus"></tatool-stimulus>

  <tatool-input id="bottomExecutable" service="input" userInput="inputAction(input, timing, event)">
    <tatool-key code="ArrowLeft" response="left"></tatool-key>
    <tatool-key code="ArrowRight" response="right"></tatool-key>
  </tatool-input>
</tatool>

We've introduced the tatool-input element on line 4, provided the predefined id bottomExecutable which will make sure the input will be displayed at the bottom of the screen. We again provide a service attribute and give it the value input, which we'll be linking to the tatoolInputService of our Executable later on. The last attribute userInput expects a function name with the three parameters input, timing, event. This is the function that should be called once the user has pressed a key or clicked on a button.

Similar to the tatool-stimulus element we're going to extend our Executable service now by adding a new dependency to tatoolInputService. See the updated code below.

myExecutable.service.js

tatool.factory('myExecutable', [ 'tatoolExecutable', 'tatoolStimulusService', 'tatoolInputService',
  function (tatoolExecutable, tatoolStimulusService, tatoolInputService) {  
    
    var MyExecutable = tatoolExecutable.createExecutable();

    MyExecutable.prototype.init = function() {
      var promise = tatoolExecutable.createPromise();

      this.counter = 0;
      this.tatoolStimulus = tatoolStimulusService.createStimulus(this.stimuliPath);
      this.tatoolInput = tatoolInputService.createInput(this.stimuliPath);

      var self = this;
      tatoolExecutable.getCSVResource(this.stimuliFile, true, this.stimuliPath).then(
        function(list) {
          self.stimuliList = list;
          promise.resolve();
        }, function(error) {
          promise.reject(error);
        });

      return promise;
    };

    // custom method to get the next stimulus out of our stimuliList
    MyExecutable.prototype.createStimulus = function() {
      var stimulus = tatoolExecutable.getNext(this.stimuliList, this.counter);
      this.tatoolStimulus.set(stimulus);
      this.counter++;
    };

    MyExecutable.prototype.stopExecution = function() {
      tatoolExecutable.stop();
    };

    return MyExecutable;
  }]);

We've added the new dependency of tatoolInputService on line 1/2. On line 11 we create a new input service and provide the only required parameter stimuli path, which would be used in the case we wanted to display images on our keys.

And finally we're going to extend our controller which with the functionality to listen to user input from the template.

myExecutable.ctrl.js

tatool.controller('myExecutableCtrl', [ '$scope', 'service',
  function ($scope, service) {

    $scope.stimulus = service.tatoolStimulus;
    $scope.input = service.tatoolInput;
    
    $scope.start = function() {
      service.createStimulus();

      service.tatoolInput.show();
      service.tatoolInput.enable();
      
      service.tatoolStimulus.show();
    };

    $scope.inputAction = function(input, timing, event) {
      service.tatoolInput.disable();
      service.tatoolStimulus.hide();
      service.stopExecution();
    };

  }]);

On line 5 we assign the tatoolInput service we've created in our Executable service to the $scope variable input. If you remember input was the value of the service attribute we've provided in the template. Now we've bridged the gap and the tatoolInput service is linked to our template.

All variables/functions assigned to the $scope object will be accessible in the Executable template.

As part of our start() method on line 10/11 we add a call to display our input buttons on screen with the show() method and activate the user input by calling the enable() method. The show() method is optional as you might choose not to display any buttons if the user has to respond by keyboard only. Finally on line 16 we add the function inputAction() to the $scope service. Within the function we make sure that the user input is deactivated by calling the method disable() and also hide our stimulus by calling the method hide(). On line 19 we call our stopExecution() method which will make sure, that once a key is pressed, the current execution is stopped.

Let's change our Module to execute our Executable several times (e.g. 3 times) instead of once. When we now execute our Module, we should be able to press the left/right arrow key to go to the next stimulus.
