Stimulus

A Stimulus is represented as a simple JavaScript object in Tatool. In order to use your stimuli objects with some of the services offered by Tatool (e.g. Tatool Stimulus, Tatool Grid), we recommend using the following naming conventions for your properties.

By following this guidelines you can also make sure that someone else can quickly understand the structure of your stimuli files.

Common Stimulus Properties

Property Values Description
stimulusValue [String] The value to be displayed on screen. This could either be a text or an image filename.
stimulusValueType text/image/circle/square Defines the type of the value in order for Tatool to know how to handle the display.
stimulusType [String] Used to group stimuli into a specific category (e.g. congruent, incongruent).
stimulusClass [String] A CSS class name that can be applied to a stimulus.
stimulusValueColor [Color name/Color hex] A generic field to be used to set the color of a text stimulus.
correctResponse [String] The expected correct response to a stimulus which can be used to compare to the given response.
gridPosition [Integer] Indicates the position of a stimuli inside a Tatool Grid.
keyCode [KeyCode] Indicates the keyCode associated with the correct response for this stimulus.
keyLabel [String] An alternate label to display on the input key. Can either be text or an image filename.
keyLabelType text/image The internal Tatool key label type in order to control the display type.
