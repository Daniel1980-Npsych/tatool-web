Display Stimuli

This section will introduce you to a simple way to display your stimuli on screen. Tatool provides some helper services that makes displaying stimuli easy.

It's time we get to edit at our Executable template, which is represented as a HTML file. Until now our template only contained the mandatory tatool element. This element represents a Directive, which couples it to some specific code in the background. Instead of you having to add your text or image elements with HTML tags, we're going to use another element called tatool-stimulus which takes care of this for us. Our new template code should now look like this.

myExecutable.html

<tatool>
  <tatool-stimulus id="centeredExecutable" service="stimulus"></tatool-stimulus>
</tatool>

We added the id attribute with the value centeredExecutable on line 2 as a simple way to position the stimulus at the center of the screen. If you wish to do any other positioning, you can simply assign your own id or class and define the necessary CSS yourself. As you can see, we've also added an attribute service to our tatool-stimulus element on line 2 and provided the value stimulus. This will allow us to create an interface for our Executable service and controller to access the template in the next step.

The value stimulus is currently not referencing anything. Let's change this now by extending our Executable service with new logic to create a stimuls service and setting a stimulus.

myExecutable.service.js

tatool.factory('myExecutable', [ 'tatoolExecutable', 'tatoolStimulusService',
  function (tatoolExecutable, tatoolStimulusService) {  
    
    var MyExecutable = tatoolExecutable.createExecutable();

    MyExecutable.prototype.init = function() {
      var promise = tatoolExecutable.createPromise();

      this.counter = 0;

      this.tatoolStimulus = tatoolStimulusService.createStimulus(this.stimuliPath);

      var self = this;
      tatoolExecutable.getCSVResource(this.stimuliFile, true, this.stimuliPath).then(
        function(list) {
          self.stimuliList = list;
          promise.resolve();
        }, function(error) {
          promise.reject(error);
        });

      return promise;
    };

    // custom method to get the next stimulus out of our stimuliList
    MyExecutable.prototype.createStimulus = function() {
      var stimulus = tatoolExecutable.getNext(this.stimuliList, this.counter);
      this.tatoolStimulus.set(stimulus);
      this.counter++;
    };

    MyExecutable.prototype.stopExecution = function() {
      tatoolExecutable.stop();
    };

    return MyExecutable;
  }]);

We've added a new dependency to the tatoolStimulusService on line 1/2. Remember that you always need to define a dependency twice, once as a string in the array and once as a parameter in the main function. On line 9 we defined a counter variable which will help us to navigate through the list of stimuli and should always be reset at the beginning of a Session to 0 (remember the first position of an Array is 0). On line 11 of our init() method we're creating a stimulus service and store it in the variable this.tatoolStimulus. The method createStimulus() expects one parameter: the stimuli path to locate the underlying stimuli files (e.g. images). We're going to provide our Property stimuliPath again, as that's where we stored our image files.

On line 26-30 we've introduced a new custom method called createStimulus() which is using the tatoolExecutable helper function getNext() on line 27 to get a stimulus at a specific position of our list. The method expects a list of stimuli (array) as a first parameter and a custom counter as the second parameter. On line 28 we set the returned stimulus to our stimulus service. Finally, on line 29 we increment our internal counter by 1 to make sure that we access the next stimuli, the next time the createStimulus method gets called.

That's it for the changes needed in our Executable service. If you remember, we've created the tatoolStimulus service which we're going to use in our template, but currently these two are not linked. That's when our controller comes into play. The controller will bridge the gap to make our tatoolStimulus service available in the template as the variable stimulus. At the same time the controller is also the place where we want to initiate the creation of a stimuli and trigger the actual display.

myExecutable.ctrl.js

tatool.controller('myExecutableCtrl', [ '$scope', 'service',
  function ($scope, service) {

    $scope.stimulus = service.tatoolStimulus;
    
    $scope.start = function() {
      service.createStimulus();

      service.tatoolStimulus.show();
      //service.stopExecution();
    }

  }]);

On line 4 we assign the tatoolStimulus service we've created in our Executable service to the $scope variable stimulus. If you remember stimulus was the value of the service attribute we've provided in the template. Now we've bridged the gap and the tatoolStimulus service is linked to our template.

All variables/functions assigned to the $scope object will be accessible in the Executable template.

On line 7 we call our custom method createStimulus() of our Executable service, which will take care of setting the current stimulus. The only thing left to do now is to display the stimulus on screen. We can take care of that by calling the method show() of the tatoolStimulus service on line 9. Notice that we've commented out line 10 for now, as otherwise the Executable would finish before displaying anything on screen.

When you execute your Module after these changes, one stimulus should be displayed at the center of the screen. The Module will stop and wait afterwards as we've commented out the call to our stopExecution() method.
