Stroop Task

The Stroop Task allows you to apply the Stroop effect paradigm.

  • Properties
  • Stimuli File
  • Download

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task.

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images)
showKeys Boolean true No Shows or hides defined input keys
timerEnabled Boolean false No Enables or disabled timer to wait for user feedback before stopping
timerDuration String 2000 No Timer duration in ms to wait for user feedback before stopping
randomisation String (none) No Defines the randomisation criteria. If no value is set no randomisation is applied. The following values are allowed:
full: fully randomises stimuli (no duplicates until all stimuli are shown once)
full-condition: randomise with even random distribution of stimulusType (duplicates allowed)

Stimuli File

This task requires a Stimuli File to run. If you're looking to randomise your stimuli, you'll only have to provide every possible combinations once. By setting the number of Iterations you can control the number of Trials. The Stimuli File should contain the following columns.

Column Name Column Values Mandatory Column Description
stimulusValue (String)/(Image Filename) Yes Value to display on screen. Can either be a text or an image.
stimulusValueType text/image/circle/square Yes The stimulus value type to identify how to display the value.
stimulusValueColor (String)/(Hex Color) Yes The color used for text/circle/square stimulusValueType.
stimulusType (String) Yes Categorize stimulus according to these groups (usually congruent, incongruent, neutral) but anything is allowed.
correctResponse (String) Yes The correct response for this stimulus.
keyCode (KeyCode) Yes The correct keyCode to press from the list of available KeyCodes (see the Tatool Input Service for more details).
keyLabel (String)/(Image Filename) No An alternate label to display on the input key. Can either be text or an image filename.
keyLabelType text/image No The key label type to identify how to display the value.

Example

stimulusValue;stimulusValueType;stimulusValueColor;stimulusType;correctResponse;keyCode
red;text;red;congruent;red;ArrowLeft
green;text;green;congruent;green;ArrowDown
blue;text;blue;congruent;blue;ArrowRight
red;text;green;incongruent;green;ArrowDown
blue;text;green;incongruent;green;ArrowDown
green;text;red;incongruent;red;ArrowLeft
blue;text;red;incongruent;red;ArrowLeft
green;text;blue;incongruent;blue;ArrowRight
red;text;blue;incongruent;blue;ArrowRight
red;circle;red;neutral;red;ArrowLeft
green;circle;green;neutral;green;ArrowDown
blue;circle;blue;neutral;blue;ArrowRight
