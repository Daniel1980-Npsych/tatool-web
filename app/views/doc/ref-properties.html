Properties

There are two different types of Properties available in Tatool. This page describes the Properties that can be customized with the help of the Module Editor. If you're looking for Module or Session Properties as part of your Executable/Handler code, take a look at the reference page of the dbUtils service instead.

A Property is defined as a key-value pair which is mostly used to make your Executables more generic and easier to work with. The key of a Property is a property name, and the value can be a given value of a specified type (String, Array, Resource, Path, Boolean, Array etc.). For example, defining (hardcoding) the name of a stimuli file in your task would mean you'd have to write a new Executable each time you'd like to use the same paradigm just with different stimuli. Instead, you can create a Property called stimuliFile of the type String, which allows for reusing the Executable by simply replacing the name of your new stimuliFile quickly within the Tatool Module Editor.

Properties that have been added with the Module Editor can be accessed in your Exectuable Service as properties of the current object (this) with the propertyName. The following line would print out the details of the Resource Property stimuliFile to the console:

console.log(this.stimuliFile);

Property Types

The following property types are available within Tatool and can be customized with the help of the Module Editor.

String
Allows you to provide a short text, letter or digit to be used within the Executable code. A String property is represented as a String object in JavaScript and can therefore be used according to its rules.

var stringProperty = 'Please answer the question.';
var anotherStringProperty = '250';

Boolean
Allows you to provide true or false value (1/0) to be used within the Executable code. A Boolean property is represented as a simple JavaScript object with the following structure.

var booleanProperty = {
  propertyType: 'Boolean',
  propertyValue: true
};

Resource
Allows you to provide a specific file resource to be used within the Executable code (e.g. stimuli file). A Resource property is represented as a simple JavaScript object. The required properties of the object differ between project (tatool internal folders) and external (web address) resources.

Project Resource:
var projectResourceProperty = {
  propertyType: 'Resource',
  project:{
    name: 'tatool-stimuli',
    access: 'public'
  },
  resourceName: 'myFile.csv',
  resourceType: 'stimuli'
};

// will be resolved to the local resource {path to projects folder}/public/tatool-stimuli/stimuli/myFile.csv
External Resource:
var externalResourceProperty = {
  propertyType: 'Resource',
  project:{
    name: 'External Resource',
    access: 'external'
  },
  resourceName: 'http://www.example.com/file.csv'
};

Path
Allows you to provide a path resource to be used within the Executable code (e.g. path to access stimuli images). A Path property is represented as a simple JavaScript object. The required properties of the object differ between project (tatool internal folders) and external (web address) paths.

Project Path:
var projectPathProperty = {
  propertyType: 'Path',
  project:{
    name: 'tatool-stimuli',
    access: 'public'
  },
  resourceType: 'stimuli'
};

// will be resolved to the local path {path to projects folder}/public/tatool-stimuli/stimuli/
External Path:
var externalPathProperty = {
  propertyType: 'Path',
  project:{
    name: 'External Resource',
    access: 'external'
  },
  resourcePath: 'http://www.example.com/'
};

Array (String)
Allows you to provide a string array to be used within the Executable code (e.g. list of image names). An Array (String) property is represented as a simple JavaScript object with the following structure.

var arrayStringProperty = {
  propertyType: 'ArrayString',
  propertyValue: [
    'a',
    'b',
    'c'
  ]
};

Array (Resource)
Allows you to provide a resource array to be used within the Executable code (e.g. list of html files). An Array (Resource) property can contain up to n Resources. An Array (Resource) property is represented as a simple JavaScript object. The required properties of the object differ between project (tatool internal folders) and external (web address) resources.

Project Array (Resource):
var projectArrayResourceProperty = {
  propertyType: 'ArrayResource',
  propertyValue: [
    {
      propertyType: 'Resource',
      project:{
        name: 'tatool-stimuli',
        access: 'public'
      },
      resourceName: 'instruction_01.html',
      resourceType: 'stimuli'
    },
    {
      propertyType: 'Resource',
      project:{
        name: 'tatool-stimuli',
        access: 'public'
      },
      resourceName: 'instruction_02.html',
      resourceType: 'stimuli'
    }
  ]
};
External Array (Resource):
var externalArrayResourceProperty = {
  propertyType: 'ArrayResource',
  propertyValue: [
    {
      propertyType: 'Resource',
      project:{
        name: 'External Resource',
        access: 'external'
      },
      resourceName: 'http://www.example.com/instruction_01.html'
    },
    {
      propertyType: 'Resource',
      project:{
        name: 'External Resource',
        access: 'external'
      },
      resourceName: 'http://www.example.com/instruction_02.html'
    }
  ]
};
