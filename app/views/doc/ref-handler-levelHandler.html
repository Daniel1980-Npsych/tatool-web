levelhandler

The levelHandler can be used in combination with the level status panel that can be activated for every Executable. The levelhandler provides the values for the level status panel who takes care of displaying the values on screen.

The levelHandler will always retrieve the latest level, counter, score at the beginning of every session. Within a session the counter will execute the level algorithm shown in the figure below after every execution of an Executable, regardless of whether a trial object is saved. In the case of a Dual Element, the algorithm will only be executed after one iteration has completed (e.g. primary, secondary, primary). The level status panel will be updated at the beginning of every new execution and should therefore show the current level.

Variables

The levelHandler allows to customize the following variables used by the level algorithm by adding a property to the handler definition in the Module file.

Property Description
benchmarkSampleSize integer value indicating the interval of when the level algorithm should be executed (number of trials)
benchmark float value between 0 and 1 indicating what percentage of correct answers (totalScore / maxScore) must be achieved for a level-up

Module/Session Properties

The levelHandler stores the following Module Properties to keep track of the level information across multiple sessions.

Property Description
counter Counter of how many trials have been executed. Used to keep track of when to trigger the level algorithm according to benchmarkSampleSize.
totalScore The summed up total score achieved over the last interval of trials (benchmarkSampleSize). Source of the value is the score property of the stored trial object.
maxScore The summed up maximum score possible over the last interval of trials (benchmarkSampleSize). The max score is set to 1 by default for every trial.
currentLevel The current level of this levelHandler
